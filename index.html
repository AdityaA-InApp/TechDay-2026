😲 WHAT SURPRISED US

The agent handled an edge case we never discussed — because the acceptance criteria implied it. The spec was smarter than the meeting.

💥 WHAT BROKE

One ambiguous requirement produced confidently wrong code — three times in a row. The bug wasn't in the model. The bug was in the spec.

🚫 WHAT WE'D NEVER DO AGAIN

Leave "obvious" things unwritten. If it's not in the spec, it does not exist.

Honest friction is the point — that's why you can believe the rest of this talk.

Frontier 3 · the new discipline

Loop engineering

PROMPT ENGINEERING · 2023
sharper words
CONTEXT ENGINEERING · 2025
sharper input
HARNESS ENGINEERING · 2026
sharper cage
LOOP ENGINEERING · now
sharper cycles
⏱ heartbeat — every N minutes 🗓 cron — on a schedule 🪝 hook — on an event (CI fails, PR opens) 🎯 goal — until the condition is met, then stop
What you actually engineer: a checkable termination condition · feedback sensors (tests, types, CI) · recoverable vs fatal failure · escalation to a human · memory between laps.

“I don't prompt Claude anymore.” — Boris Cherny, creator of Claude Code. The prompt became an artifact. The loop became the job.
