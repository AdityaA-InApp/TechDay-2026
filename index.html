What actually happened

## Requirements
[x] R1 — idempotent refund API

[x] R4 — reconciliation job, nightly

## Acceptance
9/9 scenarios green before merge
😲 WHAT SURPRISED US

The agent handled an edge case we never discussed — because the acceptance criteria implied it. The spec was smarter than the meeting.

💥 WHAT BROKE

One ambiguous requirement produced confidently wrong code — three times in a row. The bug wasn't in the model. The bug was in the spec.

🚫 WHAT WE'D NEVER DO AGAIN

Leave "obvious" things unwritten. If it's not in the spec, it does not exist.

Honest friction is the point — that's why you can believe the rest of this talk.
